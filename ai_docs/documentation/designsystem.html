  • Use outline style for inactive/default states
  • Use solid or colored style for active states

  • Color: #6b7280 (inactive), #2d9b6c (active/primary)

Layout Structure

Consistent layout patterns ensure predictable navigation and content organization.

Sidebar Navigation

  • Width: ~200px fixed
  • Background: #f3f4f6 (light gray)
  • Items: 16px vertical padding, 8px border-radius on hover
  • Active state: Light gray background (#f9fafb), primary green icon
  • Spacing: 8-12px between items

Main Content Area

  • Background: White (#ffffff)
  • Padding: 40-60px on all sides
  • Max width: Responsive, typically 1200-1400px
  • Cards: White background, 1px border (#e5e7eb), 8-12px border-radius, subtle shadow

Interactions & States

Smooth, intuitive interactions with clear feedback for all user actions.

Button States

  • Default: Solid background color
  • Hover: Slightly darker background (10-15% darker)
  • Active/Press: Even darker, slight scale transform
  • Disabled: Reduced opacity (50-60%), cursor: not-allowed
  • Transition: all 0.2s ease

Input States

  • Default: 1px border (#d1d5db)
  • Focus: Primary green border (#2d9b6c), subtle green shadow ring
  • Error: Red border, error message below
  • Disabled: Gray background, reduced opacity

Hover Effects

  • Cards: Subtle shadow increase, slight border color change
  • Links: Underline, color darkening
  • Table rows: Light gray background (#f9fafb)
  • Icons: Color change to primary green

Accessibility Notes

Design system follows WCAG 2.1 AA standards for inclusive, accessible experiences.

  • Color Contrast: All text meets minimum 4.5:1 contrast ratio against backgrounds
  • Focus States: Clear visible focus indicators on all interactive elements
  • Touch Targets: Minimum 40x40px for all clickable elements (buttons, links)
  • Alt Text: All icons paired with text labels or aria-labels
  • Keyboard Navigation: All functions accessible via keyboard
  • Screen Readers: Semantic HTML structure, proper heading hierarchy

Usage Guidelines

‚úì Do

  • Use the primary green (#2d9b6c) for main CTAs and active states
  • Maintain consistent spacing using the 4px grid system
  • Use appropriate border radius for component type
  • Keep text hierarchy clear with proper sizing and weight
  • Apply subtle shadows for elevation and depth
  • Ensure sufficient color contrast for readability

‚úó Don't

  • Use colors outside the defined palette without consultation
  • Apply inconsistent spacing or random pixel values
  • Over-use bold text or uppercase styling
  • Create custom shadow styles; use predefined elevation system
  • Mix different border radius values within the same component group
  • Compromise accessibility for visual preference
